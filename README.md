# Crypto-Project

A **Premium Dark Mode** crypto dashboard built with **Vite + React + TypeScript + Tailwind CSS**.

> ⚠️ **Note about the original request**
> The task asked to scaffold files "based on the code generated by ChatGPT", but the
> code block in the request was left as an empty placeholder
> (`[TU VLOŽ KÓD, KTORÝ TI DAL CHATGPT]`). Since no architecture code was provided,
> this repository contains a clean, sensible default architecture for a crypto
> dashboard that fully implements the requested design. Drop in / adapt your own
> generated code as needed — the folder structure and import conventions are ready
> for it.

## Design — Premium Dark Mode

The requested design is implemented as Tailwind tokens (`tailwind.config.js`):

- **App background** `#0A0A0A` → `bg-base`
- **Card background** `#141414` → `bg-card`
- **Cards** use `border border-neutral-800` and `rounded-2xl` (see `src/components/ui/Card.tsx`)
- **Header** uses a subtle glass effect with `backdrop-blur-xl` over `bg-base/70`
- **Responsive** layout: collapsible sidebar (drawer on mobile, fixed on desktop),
  responsive grids, and a table that becomes a card list on small screens.

## Getting started

```bash
npm install
npm run dev      # start the dev server
npm run build    # type-check + production build
npm run preview  # preview the production build
```

## Project structure

```
src/
├── main.tsx                 # React entry point
├── App.tsx                  # App shell (Layout + page)
├── index.css                # Tailwind layers + base styles
├── types/                   # Shared TypeScript types
├── data/                    # Mock market data (swap for a real API)
├── lib/                     # Formatting + className helpers
├── hooks/                   # useMarketData (data access layer)
├── pages/                   # DashboardPage composition
└── components/
    ├── layout/              # Header (backdrop-blur), Sidebar, Layout
    ├── ui/                  # Card, ChangeBadge primitives
    └── dashboard/           # StatCard, PortfolioCard, Holdings, MarketTable, Sparkline
```

### Imports

Imports use the `@/` alias mapped to `src/` (configured in both
`vite.config.ts` and `tsconfig.json`), e.g.:

```ts
import { Card } from "@/components/ui/Card";
import { useMarketData } from "@/hooks/useMarketData";
```

### Swapping in real data

`src/hooks/useMarketData.ts` is the single data-access point. Replace its mock
source with a real API (e.g. CoinGecko) — the component tree consumes the typed
shape and requires no changes.
